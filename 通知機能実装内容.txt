# 通知機能実装内容の詳細

## すでに実装した内容

### 1. データベース
- [x] 通知テーブル（notifications）の作成
  - `id`: UUID（プライマリキー）
  - `type`: テキスト（通知の種類：「いいね」、「コメント」、「フォロー」、「メンション」、「システム」など）
  - `sender_id`: UUID（通知の送信者ID、nullの場合はシステム通知）
  - `recipient_id`: UUID（通知の受信者ID）
  - `resource_type`: テキスト（関連リソースの種類：「プロンプト」、「コメント」、「プロフィール」など）
  - `resource_id`: UUID（関連リソースのID）
  - `message`: テキスト（通知メッセージ）
  - `is_read`: ブール値（既読状態）
  - `created_at`: タイムスタンプ（作成日時）
  - `updated_at`: タイムスタンプ（更新日時）

- [x] Row Level Security (RLS) の設定
  - 閲覧ポリシー：自分宛の通知のみ閲覧可能
  - 投稿ポリシー：誰でも通知を作成可能（システム通知やユーザーアクション通知）
  - 更新ポリシー：自分宛の通知のみ更新可能（既読状態の変更など）
  - 削除ポリシー：自分宛の通知のみ削除可能

### 2. フロントエンド
- [x] NotificationDropdownコンポーネントの実装
  - 通知アイコン表示
  - 未読通知カウントの表示
  - ドロップダウンの開閉機能
  - タブ切り替え機能（通知/お知らせ）

- [x] 通知一覧表示機能
  - 通知のリスト表示
  - 未読/既読の視覚的区別
  - 通知時間の表示

- [x] 通知設定画面（NotificationSettings）
  - 通知ON/OFFの設定
  - 通知タイプ別の設定

### 3. バックエンド
- [x] Supabase連携
  - 通知取得API連携
  - 通知既読設定API連携
  - リアルタイムサブスクリプション設定
